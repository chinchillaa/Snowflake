## 問題1

**データのロード（読み込み）とアンロード（書き出し）において、Snowflakeがサポートしている「ステージ」の種類はどれですか？（2つ選択）**

- [ ] Temporary Stages（一時ステージ）
    
- [x] **正解：External Stages（外部ステージ）**
    
- [ ] Permanent Stages（永続ステージ）
    
- [ ] Managed Stages（管理対象ステージ）
    
- [x] **正解：Internal Stages（内部ステージ）**
    

## 解説

✅ **Internal Stages（内部ステージ）**

✅ **External Stages（外部ステージ）**

Snowflakeは、データのロード・アンロード用に2つの主要なステージをサポートしています。

- **内部ステージ（Internal Stages）**：Snowflakeシステム内部にあるストレージ領域です。これには以下の3種類が含まれます。
    
    - ユーザーステージ（`@~`）
        
    - テーブルステージ（`@%table_name`）
        
    - 名前付き内部ステージ（`@stage_name`）
        
- **外部ステージ（External Stages）**：Amazon S3、Azure Blob Storage、Google Cloud Storageなどの外部クラウドストレージを参照します。ストレージの場所と認証情報を指定する必要があります。
    

これらは、Snowflakeのテーブルへデータを出し入れする際の中間ストレージとして機能します。

❌ 「Managed（管理対象）」「Temporary（一時）」「Permanent（永続）」といったステージは公式な分類としては存在しません。内部ステージの挙動として一時的・永続的な使い方は可能ですが、ステージの種類名ではありません。

**ドメイン:** データのロードと仮想ウェアハウス

## 問題2

**`Snowflake Cortex LLM 関数` の主な用途は何ですか？**

- [ ] 構造化データに対して機械学習モデルを実行すること。

## 解説

✅ **テキストデータの感情分析、要約、およびその他のAI/MLタスクを実行すること。**

Snowflake Cortex LLM関数は、自然言語処理（NLP）の機能をSnowflake環境内で直接利用できるように設計されています。これにより以下のことが可能になります。

- 長いテキストの要約

これらは完全なマネージドサービスであり、インフラの構築は不要です。データがSnowflakeの外部に出ないため、テキスト中心の分析やAIを活用したインサイトの抽出に最適です。

❌ 外部ステージのみに限定されるものではありません。

❌ Snowflake内部の運用やデータの複製を目的としたものではありません。

## 問題4

**Snowflakeにおける「システム定義のロール（役割）」はどれですか？**

- [ ] DATA_ENGINEER
    
- [ ] SNOWFLAKE_DBA
    
- [ ] SNOWFLAKE_ADMIN
    
- [x] **正解：USERADMIN**
    

## 解説

✅ **USERADMIN**

`USERADMIN` は、すべてのアカウントにデフォルトで作成されるSnowflakeのシステム定義ロールの1つです。ユーザーやロールを管理するために特化しており、`CREATE USER` や `CREATE ROLE` といった権限を持ちます。これはSnowflakeが推奨するRBAC（ロールベースアクセス制御）階層の一部であり、管理の全権限（ACCOUNTADMIN）を渡すことなく、アカウント管理業務を委任するために不可欠です。

❌ **DATA_ENGINEER / SNOWFLAKE_DBA / SNOWFLAKE_ADMIN**：これらは組織が独自の運用に合わせて作成する「カスタムロール」の名前としてよく使われるものであり、Snowflakeが標準で用意しているロールではありません。

**ドメイン:** IDとデータアクセス管理

## 問題5

**Snowflakeのテーブルからデータをアンロード（書き出し）する際、`COPY INTO @stage` コマンドの出力先（ターゲット）はどこになりますか？**

- [ ] 別のSnowflakeテーブル
    
- [x] **正解：内部または外部のステージ**
    
- [ ] データストリーム
    
- [ ] ローカルのファイルシステム
    

## 解説

✅ **内部または外部のステージ**

Snowflakeで `COPY INTO @stage` コマンドを使用すると、テーブルのデータを「ステージ（ファイルを一時的に保持する領域）」に書き出すことができます。出力先は以下のいずれかです。

- **内部ステージ**（例：`@my_internal_stage`）
    
- **外部ステージ**（認証情報を使って連携されたS3バケットやAzure Blob Storageなど）
    

これにより、クエリ結果をCSV、JSON、Parquetなどのファイル形式でエクスポートできます。

❌ **ローカルのファイルシステム**：SQLコマンドで直接手元のパソコンに書き出すことはできません。

❌ **別のSnowflakeテーブル**：テーブルからテーブルへコピーする場合は `INSERT INTO ... SELECT` を使用します。

❌ **データストリーム**：ストリームはデータの変更履歴（差分）を追跡する機能であり、アンロード先ではありません。

**ドメイン:** データのロードと仮想ウェアハウス

## 問題6

**Snowflakeの「ゼロコピークローン（Zero-Copy Cloning）」に関する記述として正しいものはどれですか？（2つ選択）**

- [x] **正解：オブジェクトの書き込み可能なコピー（複製）を作成する。**
    
- [ ] ストレージ容量を大幅に追加で必要とする。
    
- [ ] 元のデータストレージを物理的に複製（二重化）する。
    
- [x] **正解：即座に完了する、メタデータ（管理情報）のみの操作である。**
    

## 解説

✅ **オブジェクトの書き込み可能なコピーを作成する。**

✅ **即座に完了する、メタデータのみの操作である。**

Snowflakeのゼロコピークローンは、基礎となる実データを複製することなく、データベース、スキーマ、テーブルの即時かつ書き込み可能なクローンを作成します。データブロックそのものをコピーするのではなく、元のデータへの「ポインタ（参照情報＝メタデータ）」を作成するため、以下のような特徴があります。

- **高速**：クローンは数秒で作成されます。
    
- **効率的**：クローン後、または元のデータに変更が加えられない限り、追加のストレージは消費されません。
    
- **書き込み可能**：クローンされたオブジェクトは、元のデータとは独立して変更（更新・削除など）が可能です。
    

これにより、開発、テスト、バックアップなどの作業に非常に適しています。

❌ データストレージを物理的に複製しません。

❌ したがって、大幅な追加ストレージも必要としません。

**ドメイン:** データ保護とデータ共有

## 問題7

**Snowflakeのアクセス制御モデルにおける「ロールの階層化（Role Hierarchy）」の主な役割は何ですか？**

- [x] **正解：上位のロール（親ロール）が、下位のロールから権限を継承できるようにすること。**
    
- [ ] 仮想ウェアハウスのサイズを指定すること。
    
- [ ] SQLコマンドの実行順序を定義すること。
    
- [ ] ユーザーの種類を分類すること。
    

## 解説

✅ **上位のロール（親ロール）が、下位のロールから権限を継承できるようにすること。**

Snowflakeのロールベースアクセス制御（RBAC）モデルでは、ロールの階層化によって「権限の継承」が可能になります。

- 上位のロールは、下位のロールに付与されているすべての権限を自動的に継承（引き継ぎ）します。
    
- これにより、アクセス制御をモジュール化し、拡張しやすくすることができます。例えば、上位の管理者ロール（`SYSADMIN`）が、特定の部門のロール（`ANALYST_ROLE`）の権限を内包するといった管理が可能です。
    
- 権限を重複して付与する手間が省け、権限管理がシンプルになります。
    

❌ SQLの実行順序、ウェアハウスのサイズ設定、ユーザーの単なるカテゴリ分けを目的としたものではありません。

**ドメイン:** IDとデータアクセス管理

## 問題8

**Snowflakeのコア・アーキテクチャを構成する有効なコンポーネント（要素）はどれですか？（3つ選択）**

- [x] **正解：データベースストレージ層 (Database Storage Layer)**
    
- [ ] サードパーティ製のETLツール
    
- [ ] ユーザーインターフェース層
    
- [x] **正解：クエリ処理層 (Query Processing Layer)**
    
- [ ] オンプレミスのデータセンター
    
- [x] **正解：クラウドサービス層 (Cloud Services Layer)**
    

## 解説

✅ **クラウドサービス層**

✅ **クエリ処理層**

✅ **データベースストレージ層**

Snowflakeのアーキテクチャは、明確な役割を持つ3つの主要な層で構成されています。

❌ **ユーザーインターフェース層**：Snowsightなどの画面はありますが、アーキテクチャの根幹をなす3層構造には含まれません。

**ローカル（手元のパソコン等）のファイルシステムから、Snowflakeの内部ステージにファイルをアップロード（ステージング）するために使用されるコマンドはどれですか？**

## 問題11

**Snowflakeの「ロールベースアクセス制御（RBAC）」の重要な要素はどれですか？（3つ選択）**

- [ ] アクセス権は個々のユーザーの属性によって決定される。
    
- [ ] `ACCOUNTADMIN` のみがロールを作成できる。
    
- [x] **正解：ユーザーにはロールが割り当てられる。**
    
- [x] **正解：権限はロールに対して付与される。**
    
- [ ] 権限はユーザーに対してのみ直接付与される。
    
- [x] **正解：ロールは他のロールに付与することができる（ロールの階層化）。**
    

## 解説

✅ **権限はロールに対して付与される。**

✅ **ロールは他のロールに付与することができる（ロールの階層化）。**

✅ **ユーザーにはロールが割り当てられる。**

SnowflakeのRBACモデルは、スケーラブルで安全なアクセス管理を行うために以下の原則に基づいています。

- **権限はロールに対して付与される**：ユーザー個人に直接権限を与えるのではなく、ロール（役割）に対して権限を付与することで管理を簡素化します。
    
- **ロールの階層化**：ロールを別のロールに付与することで階層を作り、上位のロールが権限を継承できるようにします。
    
- **ユーザーへの割り当て**：ユーザーには1つ以上のロールが割り当てられ、セッション中にアクティブにしたロールに基づいてアクセス権が決定されます。
    

❌ **ユーザー属性によるアクセス決定**：これは「属性ベースのアクセス制御（ABAC）」であり、Snowflakeの主要モデルではありません。

❌ **`ACCOUNTADMIN` のみがロールを作成できる**：`SECURITYADMIN` など、適切な権限を持つ他のロールでも作成可能です。

❌ **権限はユーザーにのみ直接付与される**：Snowflakeではユーザーに直接権限を付与することは推奨されておらず、原則としてロールを介して行います。

**ドメイン:** IDとデータアクセス管理

## 問題15

**HIPAAやPCI DSSへの準拠を含む、最も包括的なセキュリティおよびコンプライアンス機能を提供しているSnowflakeのエディションはどれですか？**

- [ ] Virtual Private Snowflake (VPS) Edition
    
- [ ] Standard Edition
    
- [ ] Enterprise Edition
    
- [x] **正解：Business Critical Edition**
    

## 解説

✅ **Business Critical Edition**

Business Critical Edition（旧称：Enterprise for Sensitive Data）は、保護対象保健情報（PHI）やクレジットカード決済データなど、極めて機密性の高いデータを扱う組織向けに設計されています。以下の機能が含まれます。

**仮想ウェアハウスがスケールアップ（例：X-Small から Small へ変更）された場合、どのような主な効果がありますか？**

❌ **コストの削減**：サイズを上げると単位時間あたりのクレジット消費量は2倍になるため、コストは増大します。

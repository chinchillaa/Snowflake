## 問題2: COPY INTO ★試験頻出

COPY INTOコマンドの構文とオプションを説明せよ。

### あなたの回答

```
ロード（ステージ → テーブル）:
  COPY INTO [          ] FROM @stage_name;

アンロード（テーブル → ステージ）:
  COPY INTO @stage_name FROM [          ];

主要オプション:
  FILE_FORMAT: 
  ON_ERROR: 
  PURGE: 
  VALIDATION_MODE: 
```

### 正解・解説

<details>
<summary>クリックして確認</summary>

**ロード:**
```sql
COPY INTO my_table FROM @my_stage
FILE_FORMAT = (TYPE = 'CSV')
ON_ERROR = 'CONTINUE';
```

**アンロード:**
```sql
COPY INTO @my_stage FROM my_table
FILE_FORMAT = (TYPE = 'PARQUET');
```

**主要オプション:**
- `FILE_FORMAT`: ファイル形式（CSV、JSON、PARQUET等）
- `ON_ERROR`: エラー時の動作（CONTINUE、SKIP_FILE、ABORT_STATEMENT）
- `PURGE`: ロード成功後にステージからファイルを削除（TRUE/FALSE）
- `VALIDATION_MODE`: 検証のみ実行（RETURN_ERRORS、RETURN_n_ROWS）

</details>

### あなたの回答

```
VARIANT型とは:


JSONデータへのアクセス方法:
  ドット記法: column_name:[          ].[          ]
  ブラケット記法: column_name['[          ]']['[          ]']

VARIANT型を別の型に変換:
  column_name:key::[          ]

FLATTEN関数の用途:


PARSE_JSON関数の用途:

```

### 正解・解説

<details>
<summary>クリックして確認</summary>

**VARIANT型:** JSON、AVRO、ORC、Parquet、XMLなどの半構造化データを格納するデータ型

**アクセス方法:**
```sql
-- ドット記法
SELECT data:customer.name FROM my_table;

-- ブラケット記法
SELECT data['customer']['name'] FROM my_table;
```

**型変換:**
```sql
SELECT data:price::NUMBER FROM my_table;
```

**FLATTEN:** ネストされた配列を行に展開
```sql
SELECT f.value FROM my_table, LATERAL FLATTEN(input => data:items) f;
```

**PARSE_JSON:** 文字列をVARIANT型に変換
```sql
SELECT PARSE_JSON('{"key": "value"}');
```

</details>

### 正解・解説

<details>
<summary>クリックして確認</summary>

**用途:** ターゲットテーブルをソースデータと比較し、INSERT/UPDATE/DELETEを1文で実行（UPSERT）

**基本構文:**
```sql
MERGE INTO target_table AS target
USING source_table AS source
ON target.id = source.id
WHEN MATCHED THEN UPDATE SET target.col = source.col
WHEN NOT MATCHED THEN INSERT (col1, col2) VALUES (source.col1, source.col2);
```

**実行可能な操作:**
1. UPDATE（WHEN MATCHED）
2. DELETE（WHEN MATCHED）
3. INSERT（WHEN NOT MATCHED）

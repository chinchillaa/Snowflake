# Badge 4: 主要概念の整理

## データの分類

```
データの種類
├── 構造化データ (Structured)     → テーブル、CSV
├── 半構造化データ (Semi-structured) → JSON, XML, Parquet
└── 非構造化データ (Unstructured)   → 画像, 動画, 音声, PDF
```

## ステージオブジェクト

Snowflakeの**ステージ**はクラウドストレージ上のフォルダへの「名前付きゲートウェイ」。

| 種類 | 説明 |
|---|---|
| Internal Named Stage | Snowflake管理のストレージ上にファイルを格納 |
| External Stage | 外部クラウドストレージ (S3, GCS, Azure Blob) を参照 |

## ファイルフォーマット

ステージ上のファイルを正しく読み取るために、区切り文字やエンコーディングを定義するオブジェクト。

```
RECORD_DELIMITER = レコード(行)の区切り
FIELD_DELIMITER  = フィールド(列)の区切り
TRIM_SPACE       = 空白除去の有無
```

## 非ロードデータへのアクセスパターン

```
ステージ上のファイル
  ↓ ファイルフォーマット適用
  ↓ $1, $2 で列参照
  ↓ REPLACE(), TRIM等でクレンジング
  ↓ ビューとして定義
  → テーブルと同様にSELECT可能
```

## ディレクトリテーブル

`directory(@stage_name)` 関数で、ステージ内ファイルのメタデータを取得:

- `RELATIVE_PATH` — ファイルの相対パス
- `FILE_URL` — ファイルへの参照URL
- `LAST_MODIFIED` — 最終更新日時

## JOIN の種類

| 種類 | 用途 | 本ワークショップでの使用例 |
|---|---|---|
| `JOIN` (INNER) | 両方に存在するレコードの結合 | テーブルとディレクトリテーブルの結合 |
| `LEFT JOIN` | 左テーブルの全レコードを保持 | アップセル情報の付加 |
| `CROSS JOIN` | 全組み合わせの生成 | 商品 × サイズの全パターン生成 |

## データレイクハウス

ロード済みデータ（テーブル）と非ロードデータ（ステージ上のファイル）を**組み合わせる**ハイブリッドアプローチ。

```
データレイクハウス
├── ロード済みデータ (Tables)
│   ├── SWEATSUITS テーブル
│   └── UPSELL_MAPPING テーブル
└── 非ロードデータ (Stages → Views)
    ├── 画像ファイル → directory() → file_url
    ├── サイズ一覧 → ビュー
    ├── 商品ライン → ビュー
    └── コーディネート → ビュー
```

## 主要関数

| 関数 | 用途 |
|---|---|
| `REPLACE()` | 文字列の置換 |
| `INITCAP()` | 各単語の先頭を大文字化 |
| `CHR()` | ASCII文字コードから文字を生成 |
| `LISTAGG()` | 行の値を区切り文字で連結 |
| `COALESCE()` | NULL以外の最初の値を返す(pandasのfillna関数と同じ) |
| `GET_PRESIGNED_URL()` | ステージ内ファイルの一時URL生成 |
| `LEFT()` | 文字列の左からN文字を抽出 |

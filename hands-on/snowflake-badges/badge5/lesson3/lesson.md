# 📓 タイムゾーンの復習！

## 📓 世界のタイムゾーン

データ駆動型の組織には、世界中からデータが流入します。そのため、タイムゾーン（Time Zone）の仕組みを理解することは、データエンジニアにとって不可欠なスキルです。

**本初子午線**（Prime Meridian、経度0度）がイギリスのグリニッジを通り、その地域の時刻が歴史的に他のタイムゾーンの基準点として使われてきました。

- **GMT+3** は「グリニッジの時刻＋3時間」を意味します
- **GMT-4** は「グリニッジの時刻−4時間」を意味します

ただし、すべての国がグリニッジを基準にしたいわけではありません。そこで **UTC**（Universal Time, Coordinated — 協定世界時）が作られました。GMT+0とUTC+0は同じタイムスタンプになりますが、UTCはよりグローバルな表現です。

同じタイムゾーンは **Zulu Time** とも呼ばれます。軍事用語でZは「Zulu」と読むためです。

> このコースでは **UTC** 標準を使用します。
>
> 覚えておくべき略語：
> - **NTZ** = No Time Zone（タイムゾーンなし）
> - **LTZ** = Local Time Zone（ローカルタイムゾーン）

## 📓 UTCタイムスタンプの保存形式

| 部分 | 意味 |
|------|------|
| YYYY | 4桁の年 |
| MM | 2桁の月 |
| DD | 2桁の日 |
| HH | 時間（通常24時間制） |
| MI | 2桁の分 |
| SS.SSS | 2桁の秒と3桁のミリ秒 |
| +/- | UTC+0からのオフセットの方向 |
| HH:MI | UTCからのオフセット時間と分 |

## 🥋 現在のワークシートのタイムゾーンを変更する

現在のタイムゾーンを確認し、いくつかのタイムゾーンに切り替えてみましょう：

```sql
SELECT current_timestamp();

ALTER SESSION SET timezone = 'UTC';
SELECT current_timestamp();

ALTER SESSION SET timezone = 'Africa/Nairobi';
SELECT current_timestamp();

ALTER SESSION SET timezone = 'Pacific/Funafuti';
SELECT current_timestamp();

ALTER SESSION SET timezone = 'Asia/Shanghai';
SELECT current_timestamp();

SHOW PARAMETERS LIKE 'timezone';
```

> Snowflakeで使用できるタイムゾーン名はIANAリストに基づいています：
> https://data.iana.org/time-zones/tzdb-2021a/zone1970.tab

> 「**スキーマオンリード**（Schema-on-Read）」という用語を目にすることがあるかもしれません。ここでまさにスキーマオンリードが機能しています。**VARIANT** カラムには何でもロードでき、ロード後に異なる方法で解析（読み取り）できます。

フィルタリングの例：

```sql
SELECT *
FROM AGS_GAME_AUDIENCE.RAW.LOGS
WHERE AGENT IS NULL;

SELECT
    RAW_LOG:ip_address::text AS IP_ADDRESS,
    *
FROM AGS_GAME_AUDIENCE.RAW.LOGS
WHERE RAW_LOG:ip_address::text IS NOT NULL;
```

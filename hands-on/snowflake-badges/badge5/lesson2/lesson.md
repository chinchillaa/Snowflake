## 📓 ソースファイルは必ず探索する

Kishoreはとても好奇心旺盛で、それがSQLレポート作成の仕事に役立っています。好奇心はデータエンジニアを目指す上でも重要です。

> **Notepad++** は、タブや段落記号などの構造文字を表示できる優れた無料テキストエディタです。Macユーザーには **BBEdit**、**Sublime Text**、**CotEditor** などが使われています。

---

## 📓 クリエイティブなロールを使おう

データエンジニアはほとんどのタスクで **SYSADMIN** ロールを使います。**SYSADMIN** はクリエイティブなロールであり、データエンジニアはクリエイティブな作成者です。このワークショップでは、**ACCOUNTADMIN** が必要なタスク以外はすべて **SYSADMIN** を使用してください。

**SYSADMIN** が作成するすべてのオブジェクト（データベース、スキーマ、ビュー、ファイルフォーマット、ステージなど）を所有する必要があります。

> **強力すぎるロールで作成しない：** ほとんどの企業のデータエンジニアは **ACCOUNTADMIN** ロールを持っていません。迷ったら **SYSADMIN** を選びましょう！

デフォルトロールを **SYSADMIN** に設定しておくと便利です：

```sql
ALTER USER <my user name> SET DEFAULT_ROLE = 'SYSADMIN';
```

## 🥋 各属性を独自のカラムに分離するSELECT文を作成する

JSONの解析パス（PATHS）とデータ型キャスト（CASTING）をBadge 1で学んだことを思い出してください。これらのテクニックを使って、**RAW_LOG** カラムの各フィールドを個別のカラムに分離するSELECT文を構築してください。カラムの順序は問いません。日付と時刻を含むカラムは **TIMESTAMP_NTZ** にキャストしてください。

元の **RAW_LOG** カラムを最後のカラムとして含めてください。元のJSONを参照できるようにしておくのは良い習慣です。

SELECTが完成すると、**5つのカラム**が表示されるはずです。4つのカラム名は、キー/バリューペアの4つのキーに一致する必要があります。

# 🤖 DORA DNGW01

## 🥋 ワークシートで以下を実行してDORAにレポートを送信する

> **DORAのコードを変更してグリーンチェックを取得しないでください。ラボの作業を修正してください。**

GRADER関数を実行するために必要なコンテキストメニュー設定を行ってください。

```sql
SELECT GRADER(step, (actual = expected), actual, expected, description) AS graded_results
FROM (
    SELECT
        'DNGW01' AS step,
        (
            SELECT COUNT(*)
            FROM AGS_GAME_AUDIENCE.RAW.LOGS
            WHERE IS_TIMESTAMP_NTZ(TO_VARIANT(DATETIME_ISO8601)) = TRUE
        ) AS actual,
        250 AS expected,
        'Project DB and Log File Set Up Correctly' AS description
);
```

結果セクションにグリーンチェック ✅ が表示された場合のみ続行してください。⛔ が表示された場合は、現在のレッスンに戻って、見逃したステップや間違いを確認してください。

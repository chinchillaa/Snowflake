## 問題1: ノートブックでのPython変数参照

Snowflakeノートブック内のSQLセルで、`myvar` という名前のPython文字列変数を参照するための構文はどれですか？

- A) `$myvar`
- B) `'myvar'`
- C) `myvar`
- D) `{{myvar}}`

### あなたの回答

```
選択: [D]

選んだ理由・メモ:Jinja形式の{{}}でPython文字列を参照できる


```

### 正解・解説

<details>
<summary>クリックして確認</summary>

**正解: D) `{{myvar}}`**

Snowflakeノートブックでは、Jinja（ジンジャ）スタイルの構文を使用して、SQLセル内でPython変数を参照します。Jinjaとは、変数名を二重の中括弧 `{{ }}` で囲んで展開する記法で、WebフレームワークなどでPythonテンプレートとして広く使われています。

| 選択肢 | 説明 |
|--------|------|
| `$myvar` | シェルスクリプト（bash等）スタイルの変数置換。Snowflakeノートブックでは非サポート |
| `'myvar'` | 文字列リテラルとして扱われ、変数ではなく `myvar` という文字列そのものになる |
| `myvar` | 中括弧なしでは変数置換されず、テーブルの「myvar列」として解釈されエラーになる |
| `{{myvar}}` | ✅ Jinjaスタイルで正しくPython変数の値に展開される |

📖 **参照先:** まとめ/02_UI_Notebooks.md > セクション3.4（Notebooksの変数参照）

</details>

### 実際に確認してみよう

**Snowsight（UI）での手順:**
1. Snowsightにログインし、左メニューの「Projects」>「Notebooks」を開く
2. 既存のノートブックを開くか「+ Notebook」で新規作成
3. Pythonセルを追加し、`myvar = "hello"` と入力して実行
4. SQLセルを追加し、`SELECT '{{myvar}}' AS result;` と入力して実行
5. 結果に `hello` が表示されれば、Jinja構文が正しく機能していることを確認できる

**SQLでの確認:**
```sql
-- Pythonセルで変数を定義後、SQLセルでJinja構文を使って参照
-- Python側: myvar = "test_value"
SELECT '{{myvar}}' AS result;
```

GROUP BY o_orderstatus
ORDER BY count ASC LIMIT 10;

-- ORDER BY ありで最大値10件を取得

## 問題64: ウェアハウスサイズ変更のチューニングタイプ

仮想ウェアハウスのサイズをMediumからLargeに変更することは、どのタイプのチューニングに該当しますか？

- A) スケールアウト
- B) スケールダウン
- C) スケールアップ
- D) スケールイン

### あなたの回答

```
選択: []

選んだ理由・メモ:


```

### 正解・解説

<details>
<summary>クリックして確認</summary>

**正解: C) スケールアップ**

ウェアハウスのサイズ（コンピュートリソースの規模）を大きくすることを「スケールアップ」と呼び、複雑なクエリのパフォーマンス向上に寄与します。

| 用語 | 意味 |
|------|------|
| スケールアップ | ✅ ウェアハウスのサイズを大きくする（Medium → Large） |
| スケールダウン | ウェアハウスのサイズを小さくする |
| スケールアウト | クラスターの数を増やす（同時実行性の向上） |
| スケールイン | クラスターの数を減らす |


📖 **参照先:** まとめ/01_アーキテクチャ基礎.md > セクション3.4（Virtual Warehouseサイズ）
</details>

### 実際に確認してみよう

**Snowsight（UI）での手順:**
1. Snowsightの「Admin」>「Warehouses」を開く
2. 既存のウェアハウスを選択し「Edit」をクリックする
3. 「Size」のドロップダウンでサイズを変更する（Medium → Large がスケールアップ）
4. 変更を保存するとすぐにサイズが変わることを確認する

**SQLでの確認:**
```sql
-- ウェアハウスをスケールアップする（Medium → Large）
ALTER WAREHOUSE my_wh SET WAREHOUSE_SIZE = 'LARGE';

-- スケールダウン（Large → Small）
ALTER WAREHOUSE my_wh SET WAREHOUSE_SIZE = 'SMALL';

-- 現在のサイズを確認する
SHOW WAREHOUSES LIKE 'my_wh';
```

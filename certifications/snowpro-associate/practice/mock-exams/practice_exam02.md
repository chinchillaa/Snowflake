## 問題1

**Snowflakeのアーキテクチャを構成する3つの主要なレイヤー（層）は何ですか？**

- [ ] データリポジトリ、コンピュートクラスター、メタデータ管理
    
- [ ] 認証、アクセス制御、クラウドサービス
    
- [ ] データベースストレージ、ODBCドライバー、仮想ウェアハウス
    
- [x] **正解：データストレージ、クエリ処理、クラウドサービス**
    

## 解説

✅ **データストレージ、クエリ処理、クラウドサービス**

Snowflakeのアーキテクチャは、データの保存、計算処理、そして管理サービスを分離した3つの独立したレイヤーで構成されています。

- **データストレージ（Data Storage）**：すべての構造化データおよび半構造化データを、圧縮された列指向（カラムナ）フォーマットで保存します。ファイルの構成やメタデータ、最適化はSnowflakeが自動で管理します。
    
- **クエリ処理（Query Processing）**：「仮想ウェアハウス」が担当する層であり、クエリの実行やデータの変換など、すべての計算処理を行います。
    
- **クラウドサービス（Cloud Services）**：メタデータの管理、ユーザー認証、アクセス制御、クエリの最適化、インフラストラクチャの調整など、システム全体の管理を行います。
    

このように各層が分離されているため、計算リソースとストレージをそれぞれ独立して拡張（スケール）させることができ、パフォーマンスの最適化と管理の簡素化が実現されています。

**ドメイン:** Snowflakeの操作とアーキテクチャ

## 問題2

**Snowflakeはどのようにクエリを処理しますか？**

- [ ] 共有ディスク型アーキテクチャを使用する
    
- [x] **正解：MPP（超並列処理）コンピュートクラスターを使用する**
    
- [ ] クラウドストレージ内のデータを最適化することで処理する
    
- [ ] サードパーティ製のコネクタを介して処理する
    

## 解説

✅ **MPP（超並列処理）コンピュートクラスターを使用する**

Snowflakeは、MPP（Massively Parallel Processing：超並列処理）コンピュートクラスターを使用してクエリを処理します。これは「仮想ウェアハウス」として実装されています。各ウェアハウスは独立した計算ノードの集まりであり、手分けして並列に処理を行うことでクエリを高速に実行します。この仕組みにより、計算リソースを柔軟に拡張し、大規模なデータ処理にも高いパフォーマンスで対応できます。

❌ **共有ディスク型アーキテクチャ**：Snowflakeは共有ディスクとシェアードナッシング（非共有）の要素を組み合わせていますが、実際のクエリ実行はMPPクラスターが行います。

❌ **クラウドストレージ内のデータを最適化する**：ストレージの最適化も行われますが、クエリの「実行」自体はコンピュート（計算）レイヤーで行われます。

❌ **サードパーティ製のコネクタ**：コネクタはデータにアクセスするためのものであり、クエリを処理する仕組みではありません。

**ドメイン:** Snowflakeの操作とアーキテクチャ

## 問題4

**Snowflakeはデータを保存するためにどのようなストレージ形式を使用していますか？**

- [ ] 行指向（行ベース）フォーマット
    
- [x] **正解：圧縮された列指向（カラムナ）フォーマット**
    
- [ ] 圧縮された行指向フォーマット
    
- [ ] 圧縮されていない行指向フォーマット
    

## 解説

✅ **圧縮された列指向（カラムナ）フォーマット**

Snowflakeは、データ分析処理に最適な「圧縮された列指向（カラムナ）フォーマット」でデータを保存します。この形式により、クエリの実行時に必要な列（カラム）だけを読み込むことができるため、パフォーマンスが大幅に向上し、ストレージコストも削減されます。さらに、強力な圧縮技術によってデータの物理的なサイズを最小限に抑えています。

❌ **行指向フォーマット / 圧縮された行指向フォーマット**：これらは通常の業務システム（OLTP）でよく使われる形式であり、大量のデータを集計する分析処理には非効率です。Snowflakeでは使用されません。

❌ **圧縮されていない行指向フォーマット**：非効率的であり、採用されていません。

**ドメイン:** Snowflakeの操作とアーキテクチャ

```
Permanentテーブル（Time Travel 90日）の総保護期間:
  Time Travel: [    ]日 + Fail-safe: [    ]日 = [    ]日

Permanentテーブル（Time Travel 1日）の総保護期間:
  Time Travel: [    ]日 + Fail-safe: [    ]日 = [    ]日

Transientテーブル（Time Travel 1日）の総保護期間:
  Time Travel: [    ]日 + Fail-safe: [    ]日 = [    ]日
```

<details>

- 重要度に応じてTime Travel期間を適切に設定

## 問題5: Zero-Copy Cloning ★試験頻出

Zero-Copy Cloningについて説明せよ。

### あなたの回答

```
Zero-Copy Cloningとは:


クローン作成時のストレージ使用量: [増える / 増えない]

構文:
  CREATE TABLE clone_table [          ] original_table;

クローン可能なオブジェクト:
  1. 
  2. 
  3. 

元テーブルとクローンの関係:
  独立性: [依存する / 独立している]
  変更の影響: 
```

### 正解・解説

<details>
<summary>クリックして確認</summary>

**Zero-Copy Cloning:** データをコピーせずにメタデータのみをコピーしてオブジェクトの複製を作成

**ストレージ使用量:** 作成直後は**増えない**（同じマイクロパーティションを参照）

**構文:**
```sql
CREATE TABLE clone_table CLONE original_table;
CREATE DATABASE clone_db CLONE original_db;
CREATE SCHEMA clone_schema CLONE original_schema;
```

**クローン可能なオブジェクト:**
1. テーブル
2. スキーマ
3. データベース

**関係:**
- 独立性: **独立している**
- 変更の影響: 一方を変更しても他方には影響しない（変更部分のみ追加ストレージが発生）

</details>

## 問題10: データ保護総合

以下の文の正誤を判定せよ。

### あなたの回答

```
1. Time Travelを0日に設定するとその機能が無効になる
   [正 / 誤]
   理由:

2. Fail-safe期間中のデータはユーザーが直接復元できる
   [正 / 誤]
   理由:

3. Zero-Copy Cloningは作成直後から追加ストレージコストが発生する
   [正 / 誤]
   理由:

4. TransientテーブルにはFail-safeが存在しない
   [正 / 誤]
   理由:
```

3. **誤** - 作成直後は**ストレージコストなし**（メタデータのみコピー）。変更時に追加発生

4. **正** - TransientとTemporaryテーブルにはFail-safeがない

5. **正** - すべてのデータがAES-256で自動的に暗号化される

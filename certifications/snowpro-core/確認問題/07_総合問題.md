```

  GRANT SELECT ON [          ] TABLES IN SCHEMA schema_name TO ROLE role_name;

```

<details>
<summary>正解・解説</summary>

### 問題12: VARIANT型

VARIANT型のデータにアクセスする方法として正しいものはどれか？（複数選択）

- A) column_name:key
- B) column_name['key']
- C) column_name.key
- D) GET(column_name, 'key')

### あなたの回答

```
選択: [        ]

型変換（キャスト）の方法:


```

<details>
<summary>正解・解説</summary>

**正解: A), B), D)**

```sql
-- ドット記法（コロン）
SELECT data:customer:name FROM my_table;

-- ブラケット記法
SELECT data['customer']['name'] FROM my_table;

-- GET関数
SELECT GET(data, 'customer') FROM my_table;

-- 型変換
SELECT data:price::NUMBER FROM my_table;
```

C) `column_name.key` は誤り（コロンが必要）

</details>

### 問題13: Secure Data Sharing

Secure Data Sharingについて正しい記述はどれか？

- A) データがコンシューマーアカウントにコピーされる
- B) コンシューマーはデータを変更できる
- C) コンシューマーのストレージコストは発生しない
- D) 通常のビューを共有できる

### あなたの回答

```
選択: [   ]

理由:


```

<details>
<summary>正解・解説</summary>

**正解: C) コンシューマーのストレージコストは発生しない**

- A) 誤り - データはコピーされない
- B) 誤り - 読み取り専用
- **C) 正しい** - ストレージはプロバイダー負担、コンピュートのみコンシューマー負担

</details>

### 問題16: Resource Monitor

Resource Monitorで設定できるアクションとして存在しないものはどれか？

- A) NOTIFY
- B) SUSPEND
- C) SUSPEND_IMMEDIATE
- D) TERMINATE

### あなたの回答

```
選択: [   ]

各アクションの動作:
  NOTIFY: 
  SUSPEND: 
  SUSPEND_IMMEDIATE: 

```

<details>
<summary>正解・解説</summary>

**正解: D) TERMINATE**

存在しないアクション。正しいアクションは:
- **NOTIFY**: 通知のみ（クエリ継続）
- **SUSPEND**: 実行中クエリ完了後に停止
- **SUSPEND_IMMEDIATE**: 即座に停止（クエリ中断）

</details>

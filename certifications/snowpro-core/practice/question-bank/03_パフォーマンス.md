### 正解・解説

<details>
<summary>クリックして確認</summary>

| キャッシュ | 保存場所 | 有効期間 | 用途 |
|-----------|---------|---------|------|
| **Result Cache** | Cloud Services Layer | 24時間 | 同一クエリの結果を再利用 |
| **Local Disk Cache** | 仮想ウェアハウス | WH稼働中 | よく使うデータのSSD保存 |

1. 同一のSQLクエリが実行される
2. 基盤データが変更されていない
3. 24時間以内

**Result Cacheが使用されない条件:**
1. 基盤テーブルのデータが変更された
2. 24時間以上経過
3. CURRENT_TIMESTAMP()等の非決定性関数を使用

### 正解・解説

<details>
<summary>クリックして確認</summary>

**マテリアライズドビュー:** クエリ結果を物理的に保存し、自動的に更新されるビュー

## 問題10: パフォーマンス総合

以下の文の正誤を判定せよ。

### あなたの回答

```
1. Result Cacheはウェアハウスが停止していても使用できる
   [正 / 誤]
   理由:

2. ウェアハウスサイズを大きくすると同時実行クエリ数が増える
   [正 / 誤]
   理由:

3. クラスタリングキーは自動的に最適な列が選択される
   [正 / 誤]
   理由:

4. マテリアライズドビューは複数テーブルのJOINをサポートする
   [正 / 誤]
   理由:

5. Local Disk Cacheはウェアハウスを停止すると消える
   [正 / 誤]
   理由:
```

### 正解・解説

<details>
<summary>クリックして確認</summary>

1. **正** - Result CacheはCloud Services Layerに保存されるため、ウェアハウス不要

2. **誤** - サイズアップは個々のクエリ速度向上。同時実行数には**マルチクラスタ**

3. **誤** - クラスタリングキーは**手動で指定**が必要

4. **誤** - マテリアライズドビューは**単一テーブル**からのみ作成可能

5. **正** - Local Disk Cacheはウェアハウスのローカルストレージを使用するため、停止で消失

</details>

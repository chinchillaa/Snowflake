# badge4 学習まとめ更新スキル

## 目的

`/badge4/` 配下のレッスン教材（`lesson.md` や `.sql` ファイル）を読み込み、学習内容のまとめファイルを最新状態に更新する。

## 対象ファイル構成

### ソース（読み込み対象）

```
/badge4/
├── lesson1/  ... lesson10/   ← 各レッスンの教材
│   ├── lesson.md              ← レッスン本文（存在する場合）
│   └── *.sql                  ← SQLスクリプト群
```

### 生成・更新対象

```
/badge4/
├── summary.md                 ← 全体概要（レッスン一覧・オブジェクト一覧）
├── key_concepts.md            ← 主要概念の横断整理
├── sql_reference.md           ← 主要SQLリファレンス
├── lesson2/summary.md         ← Lesson2 学習まとめ
├── lesson3/summary.md         ← Lesson3 学習まとめ
├── lesson4/summary.md         ← Lesson4 学習まとめ
└── lessonN/summary.md         ← 新規レッスンが追加された場合
```

## 実行手順

### Step 1: ソースの全量スキャン

1. `ls -R /badge4` でディレクトリ構成を確認する
2. 各 `lessonN/` ディレクトリ内の `lesson.md` と全 `.sql` ファイルを読み込む
3. 既存の `summary.md` ファイルがあれば読み込み、差分を把握する

### Step 2: 各レッスンの summary.md を更新

各レッスンについて、以下の構造で `lessonN/summary.md` を作成または更新する:

```markdown
# Lesson N: [テーマタイトル]

## 学習目標
[このレッスンで何を学ぶか、1〜2文]

---

## 学んだこと

### 1. [トピック1]
[説明 + 必要に応じてSQLコードブロック]

### 2. [トピック2]
...

---

## DORA チェック: [DLKW番号]
[検証内容の簡潔な説明]
```

**ルール:**
- `lesson.md` の内容を主軸にし、SQLファイルで補完する
- SQLコードは重要な例のみ含め、冗長にしない
- 日本語で記述する
- 技術用語は初出時に英語を括弧で併記する

### Step 3: key_concepts.md を更新

全レッスンから抽出した概念を横断的に整理する:

- データの分類体系
- Snowflakeオブジェクトの説明（ステージ、ファイルフォーマット等）
- アクセスパターン（非ロードデータへのクエリ手法）
- JOIN の種類と使い分け
- 主要関数の一覧テーブル
- 新しいレッスンで登場した概念があれば追記する

### Step 4: sql_reference.md を更新

全レッスンのSQLを機能カテゴリ別に整理する:

- DDL（CREATE DATABASE, SCHEMA, TABLE, VIEW, FILE FORMAT）
- ステージ操作（LIST, SELECT from stage）
- ビュー定義（ステージ上ファイルを参照するビュー）
- JOIN / CROSS JOIN
- 集約・文字列関数の活用例
- 新しいSQL文が追加されたら該当カテゴリに追記する

### Step 5: summary.md（全体概要）を更新

- ワークショップの概要説明
- レッスン構成テーブル（新規レッスンがあれば行を追加）
- キーワード一覧（新しい概念があれば追加）
- 作成した主なオブジェクト一覧テーブル（新規オブジェクトがあれば追加）

## 記述ルール

### 全般
- 日本語で記述する（「です・ます調」ではなく「体言止め・常体」を基本とする）
- マークダウンテーブルを積極的に活用する
- SQLコードブロックは ` ```sql ` で囲む
- 既存の記述スタイルを踏襲し、一貫性を保つ

### 差分更新の方針
- 既にまとめ済みのレッスン（lesson.md や SQL に変更がない場合）は更新しない
- 新しいレッスン（lesson.md や SQL が追加されたレッスンディレクトリ）のみ追加・更新する
- 既存のレッスンに SQL ファイルが追加された場合は、該当レッスンの summary.md に反映する
- 全体ファイル（summary.md, key_concepts.md, sql_reference.md）は、新規レッスンの内容を追記する形で更新する

### コンテンツがないレッスンの扱い
- `.gitkeep` のみのディレクトリはスキップする
- `lesson.md` がなくても `.sql` ファイルがあればSQLの内容からまとめを生成する

## 注意事項

- ソースにない内容を推測で追加しない
- DORA チェック用SQL（`GRADER` 関数を使うSQL）は検証内容の説明のみ記載し、SQL全文は含めない
- 各ファイルの更新後、既存ファイルとの整合性を確認する
